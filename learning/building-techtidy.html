CASE STUDY

Building TechTidy

By Rachel Ryan • Founder, TechTidy

TechTidy didn't begin as a web development project. It began with a simple question: How can I help people feel more confident using technology?

Why I Built It

Throughout my career I've worked in technical support, systems administration, education, and technical enablement. One thing became obvious: People rarely struggle because technology is impossible. They struggle because nobody teaches it in a practical, approachable way.

I wanted to create a place where AI, digital literacy, productivity, and modern technology could be explained clearly—without hype, unnecessary complexity, or technical jargon. That became TechTidy.

Learning by Building

Rather than hiring a developer, I decided to build the platform myself. The project became an opportunity to learn modern web technologies while creating something useful. Every page, component, and resource became part of my own learning journey.

Technologies Used

  • HTML5
  • CSS3
  • JavaScript
  • Git
  • GitHub
  • GitHub Pages
  • Visual Studio Code
  • Responsive Design
  • Accessibility Principles
  • AI-assisted development

Version Control with Git & GitHub

One of the biggest skills I developed while building TechTidy was using Git and GitHub. Rather than treating version control as something only developers use, I learned how it supports experimentation, iterative improvement, and safe deployment. Each update became an opportunity to practice modern development workflows while maintaining a live production website.

Designing for Real People

Technology education isn't only about information. It's also about reducing cognitive load. Throughout the project I continuously refined:

  • Typography
  • Spacing
  • Navigation
  • Colour contrast
  • Mobile responsiveness
  • Content hierarchy
  • User experience

The goal wasn't simply to make the website attractive. It was to make learning feel easier.

Using AI as a Development Partner

AI played an important role throughout the project. Not by replacing the work—but by accelerating learning.

I used AI to:

  • Brainstorm ideas
  • Debug HTML, CSS and JavaScript
  • Improve accessibility
  • Refine copywriting
  • Explore different design approaches
  • Review UX decisions
  • Challenge assumptions

Every technical decision remained my own. AI helped me learn faster, iterate more efficiently, and spend more time thinking about the user experience instead of searching documentation for every small problem.

Accessibility Matters

One of the most rewarding parts of the project was learning how small accessibility improvements can make technology more usable for everyone. Throughout the site I incorporated:

  • Semantic HTML
  • Keyboard navigation
  • Accessible colour contrast
  • Responsive layouts
  • Meaningful headings
  • Clear navigation
  • Reduced visual clutter

Accessibility isn't a feature. It's part of good design.

What I Learned

  • Complex technology becomes approachable when it's explained clearly.
  • Building is one of the fastest ways to learn.
  • Iteration consistently produces better results than perfection.
  • Good user experience comes from listening, testing, and refining.
  • AI is most valuable when it strengthens human thinking—not replaces it.

Looking Ahead

TechTidy continues to evolve. Future projects will expand into curriculum design, technical enablement resources, AI literacy, workflow automation, and practical learning experiences that help individuals and organizations confidently adopt technology. The website is more than a portfolio. It's an ongoing record of what I'm learning, building, and sharing.

Technology should help people become more capable—not more dependent.

That's the philosophy behind TechTidy, and it's the principle that guides every project I build.
